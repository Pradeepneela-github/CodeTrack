[Git & GitHub Concepts – Simplified: --> Git: Git is like a security camera for your code. It records every change made—who did it, when, and what exactly was changed. --> GitHub: A cloud-based platform where you store Git repositories online. It’s like Google Drive for code, with collaboration features. --> Repository (Repo): Think of it like a project folder that stores your code and its entire history. Git tracks changes inside this folder. --> git status: Shows what’s going on—like asking, “What files did I change? What’s ready to save?” --> Staging (git add): Before saving changes, you “stage” them. It’s like putting items in a shopping cart before checkout. --> Commit (git commit): This is the actual save. You take a snapshot of your staged changes with a message explaining what you did. --> History (git log): Shows all past commits—who made them, when, and what changed. Like scrolling through your project’s timeline. --> Branching (git branch): Imagine you’re writing a book and want to try a new ending. You create a branch to experiment without touching the original. --> Merging (git merge): Once your changes are ready, you merge your branch back into the main one—like combining your new ending with the original story. Real-Life Debugging Example: Imagine your app suddenly stops working—it can’t connect to the database. You check the logs and see an error: “Connection refused”. You suspect someone changed the database settings. Command: 'git log config/database.yml' --> This shows who last edited the file and when. Command: 'git diff HEAD~1 HEAD' --> Now you see the exact change—maybe someone accidentally typed the wrong password. You didn’t write the code, but Git helps you trace the issue like a detective. Git Commands for Debugging Command What It Does Real-Life Use Case git log Shows commit history Check who changed a file and when. Useful when a config suddenly breaks. git blame Shows who last edited each line Find out who modified a specific line—like a broken database URL. git diff Shows changes between commits Compare versions to see what was added, removed, or changed. git show Shows details of a specific commit View the exact changes and message from a commit. git checkout Go back to a previous version Temporarily switch to an older version to test if the issue existed then. git bisect Helps find which commit introduced a bug Automates the search for the bad commit by testing versions step-by-step. ]
