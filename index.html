Real Life Debugging Example Using Git

  • When we work on real projects, issues can appear even if we did not write the code ourselves. Git can help us trace what changed, who changed it, and when it happened. Here is a simple example that new learners can easily understand.
  • Imagine my application suddenly stops working because it cannot connect to the database. The logs show an error saying "Connection refused". This usually means something in the database configuration might have been changed.
  • To investigate, I would start by checking the history of the configuration file.

    • Step 1: Check who modified the file

      Command: ["git log config/database.yml"]

  • This command shows the list of commits that affected this file. It helps you see who last edited it and at what time. This is very useful when a configuration that was working earlier suddenly breaks.

    • Step 2: Check what exactly changed

      Command: ["git diff HEAD~1 HEAD"]

  • This command compares the previous version of the file with the current one. It shows the exact lines that were added, removed, or updated. For example, you might discover that someone accidentally typed the wrong database password or changed the URL.
  • Even if I were not the person who made the change, Git allows me to trace the issue step by step, almost like solving a small mystery.

Common Git Commands for Debugging

Here is a quick reference table that explains the most useful Git commands for debugging in a simple way.
Git Command What It Does How It Helps in Debugging
git log Shows the commit history Helps you see who changed a file and when, useful when something suddenly stops working
git blame Shows who edited each line Helps identify who modified a specific line, such as a wrong database URL
git diff Shows differences between versions Helps you see what was added, removed, or changed between commits
git show Displays details of a specific commit Helps you review the commit message and the exact changes made
git checkout Switches to a previous version Helps you test whether the issue existed in older versions
git bisect Helps find the commit that introduced a bug Automates the process of locating the faulty commit by testing versions one by one
